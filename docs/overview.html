Infrastructure

Tech stack

Every layer chosen to run permanently at zero cost — using GitHub Student Pack credits as runway, and free-forever tiers as the long-term home. Each card includes the fallback if the primary option fails.

Hosting — Year 1

DigitalOcean Droplet

$200 GitHub Student credit funds a small droplet (1–2 vCPU, 2GB RAM) running Docker for a full year with zero setup friction.

Fallback: Azure for Students ($100 credit)
Hosting — Forever

Oracle Cloud Always Free

2 OCPU / 12GB Arm instance, genuinely free forever. Migrate here once the DigitalOcean credit runs out.

Fallback: Google Cloud e2-micro (1GB RAM, free forever)
Automation

n8n (self-hosted)

Workflow engine running every scheduled job, webhook, and integration between Telegram, the database, and the AI layer. Runs as a Docker container.

Version: latest stable via Docker image
Database

Supabase

Postgres with built-in auth, storage, and row-level security. Free tier: 500MB DB, 1GB file storage — plenty for one user's data.

Fallback: Self-hosted Postgres on the same droplet
Intelligence

Gemini API

Handles prioritization reasoning, PDF summarization, quiz generation, resume gap analysis. Generous free-tier daily request limits.

Fallback: Local Ollama model (e.g. Llama/Mistral 7B) on the droplet
Interface

Telegram Bot API

The primary way you talk to LifeOS — commands, natural language, file uploads, daily digests, inline buttons, all in one chat thread. No rate limits that matter at personal scale.

Dashboard — Later

React + Tailwind

A visual layer added once the backend is solid — charts, analytics, manual data entry when Telegram isn't convenient. Auth via Supabase Auth.

Extras

Domain + Monitoring

Free domain (1 yr) via the Student Pack. Optional: Prometheus + Grafana (both free, self-hosted) to watch droplet health and n8n execution success rate.

Data layer

Database schema, by module

One Supabase project, one schema — every module reads and writes here, which is what lets AnalyticsOS query across all of them at the end.

System processes

7 modules, one data model

Each module is independent enough to build and demo on its own, but they all read and write to the same Supabase backend — which is what makes the whole thing feel like one system instead of seven bots.

n8n pipelines

How data actually moves

Six representative workflows covering every trigger pattern you'll need — scheduled, message-triggered, and file-triggered. Everything else in the system follows one of these shapes.
